# 2026-06-29 工作日志

## 第一阶段 checkpoint

完成项目第一阶段骨架搭建。

关键决策：

- 后端先只接入 Web、Validation 和测试依赖，不提前引入 Spring Security、MyBatis Plus、Spring AI。
- 前端不使用 Element Plus，采用 Vue3 + Tailwind CSS + shadcn-vue 风格自定义组件。
- 图标包从已弃用的 `lucide-vue-next` 改为 `@lucide/vue`。
- 视觉风格保持类似 Codex 的简洁工具型产品风格。

遇到的问题：

- Spring Boot 4.1 的 `AutoConfigureMockMvc` 包名和旧版本不同。
- npm 安装 `lucide-vue-next` 时进程卡住，并提示该包已弃用。

处理结果：

- 将测试导入改为 `org.springframework.boot.webmvc.test.autoconfigure.AutoConfigureMockMvc`。
- 中断卡住的 npm 进程，改用 `@lucide/vue`。
- 用 Playwright 检查登录页和工作台的桌面端、移动端渲染。

## 第二阶段 checkpoint

完成登录与权限基础能力。

关键决策：

- 登录阶段先使用内存演示用户，不立即引入数据库和 Spring Security。
- token 使用 JDK 自带 HMAC-SHA256 实现，避免过早引入 JWT 依赖。
- 前端登录后保存 `token` 和 `user`，路由守卫根据 token 判断是否允许进入业务页面。
- 数据库表结构先以 `docs/sql/init.sql` 草稿形式沉淀，下一阶段再接 MyBatis Plus。

遇到的问题：

- Spring Boot 4 使用 Jackson 3，测试里不能继续使用旧的 `com.fasterxml.jackson.databind` 包。
- 本地 8080 上存在旧后端进程，健康检查能通，但新登录接口返回 404。

处理结果：

- 后端测试改用 JsonPath 读取登录响应中的 token。
- 重启旧后端进程后，真实 HTTP 登录接口和当前用户接口均可正常访问。
- 用 Playwright 检查登录页提交后能跳转到工作台，浏览器控制台无错误。
